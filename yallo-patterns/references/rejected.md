# YALLO Rejected Patterns — Never Use

These are explicitly banned from the YALLO brand system. Do not suggest or implement any of these regardless of context.

| Pattern | Reason |
|---------|--------|
| Honeycomb | Too decorative. Consumer-tech aesthetic. Conflicts directly with the architectural, precision positioning of YALLO. A CIO would not respond positively to this. |
| Circuit board | Cliché for IT/tech companies. Reads as a generic stock template from 2015. Undermines the premium positioning. |
| Carbon fibre | Luxury-car aesthetic. Wrong sector entirely. Sends no signal relevant to enterprise talent or architecture. |
| Photographic backgrounds | Never behind text content. Photos behind text break readability and dilute the brand voice. |
| Coloured non-brand backgrounds | Only approved surfaces are: #000000, #111111, #1F1F1F, #FFFFFF, #F9F9F7, #F4F4F2. No other background colours. |
| Two patterns on same surface | Never layer two patterns. One pattern per layout is the hard rule. |
| Pattern above 10% opacity (when text present) | Any pattern above 10% opacity when text sits above it is rejected. Readability is non-negotiable. |
| Gradient glow (Pattern C) on every slide | Pattern C is for hero sections and cover slides only — not every slide in a deck. |
