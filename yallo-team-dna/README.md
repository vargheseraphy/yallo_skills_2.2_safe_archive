# YALLO Team DNA — Skill 1

## What This Skill Does

This skill gives Claude complete intelligence about the YALLO Group team.
When loaded, Claude knows exactly who every person is, what they own,
how they work, what to delegate to them, what not to pull them into,
how content gets approved, how the ABM pipeline flows, and who has
decision authority over what.

Use this skill any time you need Claude to:
- Assign a task to the right person
- Write content in a specific person's voice
- Design a workflow or process involving the team
- Plan a campaign and know who does what
- Draft a communication on behalf of someone
- Understand the approval chain for any output
- Map the ABM pipeline from account to deal to delivery

## How to Install

### Option 1 — Install from .sip file
Upload `yallo-team-dna.sip` to your Claude skills manager.
The skill will appear as `yallo-team-dna` and trigger automatically
when you ask Claude about YALLO team tasks, content, workflows,
or delegation.

### Option 2 — Manual installation
Copy the contents of `SKILL.md` into your skills folder at:
`/mnt/skills/user/yallo-team-dna/SKILL.md`

## How to Use

Once installed, Claude will automatically use this skill when you ask
questions involving YALLO team members, content, campaigns, or processes.

**Example triggers:**
- "Write a LinkedIn post for Sumeet about EA and AI"
- "Who should I assign the newsletter to?"
- "Create a workflow for the India ABM campaign"
- "Draft an email from Abhinav to a GCC prospect"
- "Who approves the case study before it goes live?"
- "Map out the process from account identified to meeting booked"

You can also explicitly trigger it by saying:
"Using the YALLO Team DNA skill, help me with..."

## How to Update

This skill should be reviewed and updated every 3 months.

**Next review due:** June 2026

**Update process:**
1. Open `SKILL.md`
2. Answer the five review questions in Section 8
3. Update the relevant profile, map, or process section
4. Update the version number in the YAML frontmatter (e.g., 1.0.0 → 1.1.0)
5. Add a row to the Update Log table in Section 8
6. Repackage as a new `.sip` file
7. Reinstall to replace the previous version

**Five review questions (Section 8):**
1. Who has changed roles or responsibilities?
2. Any new tools actively being used?
3. Any processes not working that need redesigning?
4. Any new team members to add or departures to remove?
5. Any positioning or service changes at YALLO?
